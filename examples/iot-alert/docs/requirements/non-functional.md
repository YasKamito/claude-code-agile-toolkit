# 非機能要件 — IoTアラート

> 最終更新: 2026-03-30

---

| カテゴリ | 要件 | 目標値 | 優先度 | 備考 |
|---------|------|--------|--------|------|
| **性能** | センサーデータ取得間隔 | 5秒〜60秒（設備による） | 高 | PLCゲートウェイのポーリング間隔に依存 |
| 性能 | アラート通知遅延 | センサー値受信から通知送信まで30秒以内 | 高 | 夜勤帯の即時検知に直結 |
| 性能 | ダッシュボード表示速度 | 初期表示3秒以内（17台） | 中 | マルチ拠点（50台）時も5秒以内 |
| **可用性** | システム稼働率 | 99.5%以上（月間ダウンタイム約3.6時間まで） | 高 | 工場は24時間稼働。夜勤帯のダウンは特にリスク |
| 可用性 | 計画メンテナンス | 月1回、夜間帯に最大1時間 | 中 | 保全担当者に事前通知必須 |
| **セキュリティ** | 通信暗号化 | 全通信HTTPS/TLS 1.2以上 | 高 | BR-013 |
| セキュリティ | VPN接続 | クラウドへの接続はVPN経由のみ | 高 | 大野さん必須要件 |
| セキュリティ | 認証 | メール+パスワード（MVP）→ AD連携（v2） | 高 | パスワードポリシー: 8文字以上、英数記号混合 |
| セキュリティ | アクセスログ | 全操作をログ記録。保持期間: TBD | 高 | 大野さん必須要件。BR-014 |
| セキュリティ | APIキー管理 | PLCゲートウェイのAPIキーを安全に管理 | 高 | Secrets Manager等で暗号化保管 |
| **スケーラビリティ** | 設備台数 | MVP: 17台、v1: 50台（3拠点） | 中 | アーキテクチャで考慮。段階的に拡張 |
| スケーラビリティ | データ保持期間 | リアルタイム: 直近7日、アーカイブ: 1年 | 中 | ストレージコストとのバランス。要検討 |
| **運用・保守** | 運用主体 | CM側で運用（大野さん要件） | 高 | 監視・障害対応・バックアップ |
| 運用・保守 | バックアップ | 日次自動バックアップ | 中 | 閾値設定・ユーザー情報・アラート履歴 |
| 運用・保守 | 監視 | システム自体の死活監視・リソース監視 | 中 | CloudWatch等 |

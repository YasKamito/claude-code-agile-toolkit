## トレーサビリティ

```
業務フロー（business-flow.md）
  → ストーリー（story-mapping.md: S-xxx / M-xxx / A-xxx）
    → ビジネスルール（business-rules.md: BR-xxx）
    → 画面（screen-list.md: SCR-xxx）
      → PBI（本ファイル: PBI-xxx）
        ← ADR（docs/adr/: 技術判断）
```

## 補足

- SP目安はTシャツサイズ（S/M/L/XL）で概算。詳細見積もりはスプリントプランニングで行う
- 出所: どの要件定義成果物から導出されたか（traceability）
- このバックログは初期版。リファインメント（`/refinement check`, `/refinement ac`）で品質を高めていく

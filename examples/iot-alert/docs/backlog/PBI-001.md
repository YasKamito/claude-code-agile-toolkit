## ストーリー

開発チームとして、PLCゲートウェイからセンサーデータを定期取得・蓄積したい。なぜならダッシュボード表示・閾値判定・アラート通知の全ての土台となるデータ基盤が必要だから。

## 受入条件

- [ ] PLCゲートウェイのREST APIからセンサーデータをポーリングで取得でき、レスポンスのJSONスキーマ（device_id, sensor_id, value, timestamp）が検証される
- [ ] 取得データ（device_id, sensor_id, value, timestamp）がデータストアに保存される
- [ ] 本社工場17台分の設備データを5秒間隔（設定変更可能）でポーリングできる
- [ ] PLCゲートウェイへのAPIリクエストが3回連続でタイムアウト（10秒）またはエラーとなった場合、通信断としてログに記録される
- [ ] ゲートウェイ復旧後、バッファリングされたデータ（最大30分分）を時系列順に取り込み、欠損期間がログに記録される
- [ ] APIキーはAWS Secrets Managerで管理され、Lambda環境変数に平文で保存されない

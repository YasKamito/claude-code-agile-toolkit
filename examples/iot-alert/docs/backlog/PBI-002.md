## ストーリー

保全担当者として、センサー値が閾値を超えた場合に自動で異常レベルを���定してほしい。なぜなら経験に頼らず、データに基づいて異常に気づけるようになりたいから。

## 受入条件

- [ ] センサー値が閾値の80%以上100%未満の場合、「注意」レベルと判定される（BR-001）
- [ ] センサー値が閾値の100%以上120%未満の場合、「警告」レベルと判定される（BR-002）
- [ ] センサー値が閾値の120%以上の場合、「異常」レベルと判定される（BR-003）
- [ ] センサーデータが60秒以上途絶した場合、「異常」（センサー断）と判定される（BR-011 ※暫定値）
- [ ] 閾値は設備×センサー項目ごとに異なる値を参照する（BR-009）
- [ ] 判定結果（設備ID、センサーID、値、レベル、タイムスタンプ）が記録される
- [ ] レベル遷移イベント（正常→注意→警告→異常）が記録され、遷移前後のレベルと遷移時刻が含まれる

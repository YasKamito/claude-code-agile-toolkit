## Q1. 我々はなぜここにいるのか

### 顧客の課題
- 設備故障が**起きてから**知る状態。壊れる前に知りたい
- ベテラン保全担当者の経験と勘に依存した監視（「プレス機は音で8割わかる。でもそれを若い人に教えるのは難しい」）
- 夜勤帯は保全が1人しかおらず、異常検知が遅れるリスクが高い
- 3拠点間の状況確認が電話ベース。リアルタイムで見えない

### 放置した場合のインパクト
- 突発故障による生産ライン停止（直接的な売上損失）
- ベテラン退職・世代交代で監視ノウハウが失われる
- 夜勤帯の対応遅れによる設備損傷の拡大・安全リスク

### プロジェクトが実現する世界
- 閾値ベースの自動通知で「壊れる前に気づける」状態
- 経験に頼らず、データとアラートで若手でも対応できる仕組み
- 3拠点の設備状態を一画面で一元監視。電話不要

## Q2. エレベーターピッチ

**工場の保全担当者**が
**設備異常の早期検知と迅速な対応**に困っている。
**IoTアラート**は**設備監視・通知システム**で、
**センサーデータのリアルタイム監視と閾値ベースの自動アラート通知**を提供する。
**PLC画面の目視確認やベテランの勘と経験**とは違い、
**マルチ拠点一元監視・設備ごとの閾値カスタマイズ・Slack/メールへの即時通知**が特徴である。

### 補足
- **ターゲットユーザー**: 保全担当者（拠点別・設備種別の担当制）、工場管理部門長
- **主要な価値**: 壊れる前に気づける。夜勤帯でも確実にアラートが届く
- **差別化ポイント**: PLCに蓄積されているデータを活用し、保全担当者自身が閾値を設定・変更できる。IT部門に依頼不要
